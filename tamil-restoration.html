Tamil Classical Text Restoration System

An end-to-end research and production system for restoring word boundaries and diacritics in classical Tamil, with domain adaptation for Siddha medical manuscripts.

Restoration example

Before தானானவயிததியததிலரெண்டு After தானான வயித்தியத்தில் ரெண்டு

Problem

Classical and Agasthiyar-style Tamil is commonly digitized without word spaces or vowel diacritic marks (due to historical manuscript conventions and OCR pipelines). The text தானானவயிததியததிலரெண்டு looks nothing like its readable form தானான வயித்தியத்தில் ரெண்டு. Two characters become fifteen. Human readability collapses entirely.

The task: map unspaced, partially diacritized Tamil to correctly spaced, fully restored Tamil. At scale, across a corpus of ~14,000 sentence pairs spanning modern and classical domains.

Core Insight and Architecture

The standard approach would be seq2seq — ByT5-large as the byte-level encoder-decoder. We started there. The baseline CER was 11.95%, and we observed structural failure modes: repetition loops, leading-chunk drops, and hallucinated content. These are properties of autoregressive generation over long Tamil byte sequences, not fixable with prompt engineering or decoding tricks.

The key intellectual move was to reframe restoration as character-level insertion tagging, not generation.

Tamil text restoration is near-monotonic: the output is always the input plus insertions of spaces, virama marks, and vowel signs, occasionally with character substitutions. Given this, we replaced the seq2seq model with a tagger that assigns one discrete label to each input character:

  • K — keep as-is
  • D — delete
  • S:x — substitute with character x
  • Any of the above with a suffix string (space, virama ், vowel sign, combinations)

This makes repetition loops and content hallucination structurally impossible. The model cannot invent characters outside the label vocabulary. Output length is bounded by the input.

We derived these labels from ~14,000 aligned pairs using edit-path alignment (Levenshtein editops), collapsing all alignment decisions into a compact per-character label set.

Architecture: CharConvTagger — a 12-layer dilated 1D CNN with exponentially cycling dilation factors (1, 2, 4, 8, 16, 32, 64, 128, 1, 2, ...), 384-dim hidden, GELU activations, residual connections, LayerNorm, linear projection head. ~15M parameters. Runs on CPU at inference. Linear time over sequence length, receptive field covering ~250 characters.

Training

  • Curriculum bucketing via LengthBucketBatchSampler — grouped sequences by length to minimize padding waste, critical for long classical stanzas (up to 2,000 characters)
  • Oracle label analysis confirmed near-zero oracle CER — the tagger framework was theoretically sound before a single training run
  • Multi-seed training with logit ensembling: averaged raw logits from independent runs before decoding, providing calibration improvement without any inference overhead beyond a single model load
  • CRF head added to capture adjacent-label dependencies
  • Focal loss and class-weighting experiments for the rare substitution labels (~1% of positions)

Constrained Decoding

Even with the tagger, a subtle failure mode appeared on Agasthiyar medical Tamil: Tamil dependent vowel signs (ே, ெ, ை, etc.) appearing after word boundaries — a Unicode-valid but Tamil-grapheme-invalid sequence like சாற்றின் ேனதை.

We implemented Tamil grapheme-constrained decoding: at each position, the decoder walks the ranked label list and accepts only the highest-scoring label whose emitted segment keeps the output a valid Tamil grapheme sequence. A repair heuristic reattaches orphan marks to the nearest preceding valid consonant rather than silently dropping them.

This is a hard constraint enforced during inference, not a post-processing filter. Zero orphan vowel signs in all tested outputs after this change.

Domain Adaptation: Agasthiyar Medical Tamil

Standard fine-tuning on domain data with a fixed vocabulary fails when the domain introduces new character combinations and label patterns. We implemented vocabulary expansion without weight loss:

  • Identified new characters and label patterns from 379 Agasthiyar/Siddha medical Tamil parallel pairs
  • Expanded embedding and classification head matrices; copied all old weights by identity; initialized new rows from the nearest-code existing label
  • Fine-tuned with mixed batches (domain repeated 120×, 40,000 general-domain samples per epoch)
  • Agasthiyar validation CER improved from 8.45% → 5.14%. General-domain CER moved negligibly from 0.53% to 0.59%.

Agasthiyar validation CER: 8.45% → 5.14%. General-domain CER: 0.53% → 0.59% (negligible drift).

N-Best Reranking

For ambiguous domain text, we built a deterministic n-best reranking layer that avoids any LLM call:

  • Beam generation: Constrained beam decoder explores top-k label alternatives per position, maintaining a beam of distinct output strings with cumulative log-probabilities
  • Multi-signal scorer combining:
    • Normalized model log-probability
    • Tamil grapheme validity score (fraction of dependent signs with valid consonant context)
    • Domain lexicon coverage over 7,782 domain word forms extracted from Agasthiyar targets
    • Character 4-gram LM score trained on restored Agasthiyar targets (add-alpha smoothing)
    • Suspicious fragment penalty (long unknown words, double spaces)
    • Placeholder fragmentation penalty (prevents char-LM from preferring expanded dot sequences)
  • Lexicon repair: For candidates with high domain lexicon context score, Levenshtein nearest-neighbor search over domain vocabulary proposes token-level repairs as additional candidates — gated by domain-context threshold to prevent spurious repairs on general Tamil

Reranking improved full Agasthiyar corpus: 1.07% → 0.76% CER, 81.0% → 85.75% exact match.

What Makes This Interesting

A few things that distinguish this from standard NLP:

  1. The reframing is the contribution. Most people would train a larger seq2seq model or add beam search. Recognizing that the monotonicity of Tamil restoration makes generation the wrong abstraction — and that insertion tagging gives you hard structural guarantees — is the kind of decision that shows understanding of inductive bias.
  2. Unicode-aware constrained decoding. Tamil Unicode has strict grapheme rules (dependent signs must follow consonants). Building a decoder that respects these as hard constraints during inference — not as a post-processing filter — required understanding both the Unicode Tamil block and how label sequences compose at character boundaries.
  3. Domain adaptation without vocabulary collapse. Expanding embedding and classification matrices mid-training while preserving existing weights by character identity is not the default behavior of any standard fine-tuning library. It required deliberate surgery on the model state dict.
  4. The reranker is interpretable and deployable. The n-best reranker produces scored candidates with full diagnostics (per-signal scores, repair traces) and adds no neural model at inference time. That's a practical engineering choice: explainable, debuggable, no GPU dependency for the reranker itself.
